## ğŸ”„ Documentation Generation

### How Documentation is Generated

**Native HTML Generation (v1.1.0+):**
- No Python required
- Self-contained HTML files
- Generated by TypeScript, Go, and Rust implementations
- Each implementation creates its own docs directory

**Generation Methods:**
1. **Auto-generation:** On first dashboard server startup
2. **Manual:** Via `@dependency-buster generate_mkdocs_docs` command
3. **CI/CD:** Via JSON-RPC calls in build pipelines

**Output Formats:**
- `format=html` - Self-contained HTML (for humans)
- `format=markdown` - Markdown files (for AI agents)
- Both formats can be generated simultaneously

### Future: Auto-Updates (v2.0)

Planned features for automatic documentation updates:
- Watch for code changes and auto-regenerate
- Git hooks for pre-commit/post-commit updates
- CI/CD integration for automated generation
- Webhook support for external triggers
- Incremental updates (only changed sections)
- Version control integration
- Multi-branch documentation
- Documentation diff view

See [ROADMAP.md](./ROADMAP.md) for details.
